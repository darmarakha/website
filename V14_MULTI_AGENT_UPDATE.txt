GEMU AI v14 — Multi-Agent Sistem, Reports Cleanup, dan Apply Bertahap

Perubahan utama:
1. Menambahkan arsitektur 3 role agent di dalam satu sistem GEMU:
   - GEMU Sistem: skor, keputusan, approve owner, anti-numpuk.
   - GEMU Frontline: komunikasi natural, mode tamu, UX/tampilan.
   - GEMU Backend: PHP/JS/security/storage/database, patch, test, backup, rollback.

2. Menambahkan agent scoring 0–100:
   - Pemahaman intent: 20 poin
   - Keamanan perubahan: 25 poin
   - Kesesuaian file target: 15 poin
   - Risiko bug kecil: 15 poin
   - Test dasar lolos: 15 poin
   - UX/tampilan tidak rusak: 10 poin

3. Aturan keputusan:
   - 0–59: ditolak otomatis, tidak buat draft.
   - 60–79: hanya saran/analisis, belum layak approve.
   - 80–89: draft edit menunggu approve owner.
   - 90–100: draft kuat, tetap menunggu approve owner.

4. Apply setelah Darma klik ✓ sekarang memakai proses bertahap:
   - Lock proses agar tidak numpuk.
   - Cek hash file lama.
   - Test otomatis ulang.
   - Backup file lama.
   - Tulis file.
   - Test setelah apply.
   - Rollback otomatis jika post-test gagal.
   - Simpan log proses.

5. Laporan GEMU tidak lagi diarahkan ke root public_html.
   - Folder baru: AI/reports/
   - File lama gemu-laporan-*.html / gemu-analisis-*.html di root akan dipindah saat cleanup berjalan.
   - Auto-cleanup laporan: maksimal 10 laporan terbaru, hapus yang lebih dari 7 hari, hapus duplikat.

6. Cron juga ikut cleanup laporan dan backup.

7. UI owner ditambah panel 3 Role Agent GEMU dan ringkasan scoring visual.

Catatan aman:
- Edit website tetap tidak diterapkan sebelum Darma klik ✓.
- Tombol × menolak draft tanpa mengubah website.
- GEMU tetap tidak bisa menjamin 100% bebas error di hosting PHP/cPanel, tapi risiko diturunkan dengan backup, staged edit, hash check, test, rollback, scoring agent, dan approve owner.
