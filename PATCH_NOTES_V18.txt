GEMU AI v18 - Real Multi-Agent Hardening

Perbaikan utama:
1. Menambahkan agent decision queue terpisah dari staged edit.
   - Jika diskusi 3 role menghasilkan skor >=80 tetapi belum berupa patch file, tombol ✓/× tetap muncul di Reaksi Draft.
   - Approve decision hanya menyimpan keputusan/rencana agent; edit website tetap harus staged edit dan butuh ✓ lagi.

2. Memperkuat diskusi 3 role GEMU.
   - GEMU Frontline: komunikasi, mode tamu, UI/UX, pemahaman bahasa Darma.
   - GEMU Backend: PHP/JS/security/storage/database, patch, test, backup, rollback.
   - GEMU Sistem: skor, risiko, keputusan, approve owner, anti-numpuk.
   - Prompt agent berjalan minimal beberapa ronde sehingga tidak langsung berhenti hanya karena skor awal tinggi.

3. Scoring dibuat lebih berbasis bukti.
   - Pemahaman intent 20 poin.
   - Keamanan perubahan 25 poin.
   - Kesesuaian file target 15 poin.
   - Risiko bug kecil 15 poin.
   - Test dasar lolos 15 poin.
   - UX/tampilan tidak rusak 10 poin.

4. Reaksi owner lebih jelas.
   - Reaksi Draft sekarang menampilkan staged edit dan agent decision.
   - Tombol ✓ Setuju dan × Tolak bisa bekerja untuk keduanya.
   - Status pending agent ikut dihitung di panel memori.

5. Router keamanan owner diperbaiki.
   - Pertanyaan seperti "bisa memperbaiki sistem agar sulit dibobol?" tidak langsung dianggap perintah patch umum.
   - GEMU menjawab dengan arah security audit lokal dulu: SQL injection, XSS, upload, CSRF, header, permission, dan fungsi server berisiko.

6. Mode tamu tetap dibatasi natural.
   - Sapaan dijawab sapaan.
   - Pertanyaan "kamu siapa?" dijawab sebagai identitas GEMU dulu, bukan profil Darma.

Test yang dijalankan:
- php -l seluruh file PHP utama: lolos.
- node --check AI/gemu.js: lolos.
- Test HTTP lokal public_chat: "hallo" dan "kamu siapa?" menghasilkan jawaban natural.
- Test HTTP lokal owner security prompt: menghasilkan arahan security audit lokal, bukan audit file/storage acak.
- Test HTTP lokal agent_dialogue dengan session owner dummy: menghasilkan score 96/100 dan agent decision pending.

Catatan jujur:
Tidak ada sistem hosting PHP/cPanel yang bisa dijamin 100% tanpa bug/celah. Versi ini menurunkan risiko dengan staged edit, owner approve, agent scoring, token owner, backup, hash/test, dan decision queue.
